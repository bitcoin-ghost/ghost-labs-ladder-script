What Is This

Every transaction format needs proof that it works end-to-end: fund a UTXO with spending conditions, then spend it by satisfying those conditions. This page documents proven Ladder Script transaction round-trips — real outputs that were created, confirmed, and then successfully spent on a live network.

Each record shows both the funding and spending transaction, the witness data, and cryptographic verification that the spending conditions were correctly evaluated.

How A Round-Trip Works

Each record proves the full lifecycle of a Ladder Script UTXO — from creation to redemption.

1

Create Conditions

Design spending conditions using Ladder Script blocks — signatures, timelocks, hash preimages, covenants, or any combination. Each block is typed and deterministic.

2

Fund the Output

Build a v4 RUNG_TX transaction that creates an output with the specified rung conditions. The conditions are serialized and committed in the output. For PQ schemes, only a 32-byte PUBKEY_COMMIT is stored on-chain.

3

Spend the UTXO

Construct a spending transaction that provides a ladder witness satisfying the conditions. The evaluator checks each block in each rung — first satisfied rung wins (OR logic), all blocks within a rung must pass (AND logic).

4

Verify On-Chain

The spending transaction is broadcast, validated by consensus, and confirmed in a block. The full witness is parseable and every cryptographic check can be independently verified.
